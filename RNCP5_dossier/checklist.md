# Checklist RNCP 5

## A completer

- [ ] Respect du volume corps du rapport: 30 a 50 pages (hors garde, sommaire, annexes)
- [ ] Respect du volume annexes: maximum 30 pages
- [ ] Cahier des charges identifie: projet en entreprise ou projet en formation
- [ ] Identite et informations administratives
- [ ] CV a jour
- [ ] Presentation professionnelle
- [ ] Liste des competences du referentiel couvertes par le projet

### Cahier des charges - Contexte projet (entreprise)
- [ ] Presentation de l entreprise et du service
- [ ] Cahier des charges ou expression des besoins
- [ ] Contraintes du projet et livrables attendus
- [ ] Environnement humain, technique, objectifs de qualite

### Cahier des charges - Contexte projet (formation)
- [ ] Listes des competences mises en oeuvre dans le cadre du projet
- [ ] Expression des besoins (objectifs et limites du projet)
- [ ] Environnement technique
- [ ] Realisations permettant la mise en oeuvre des competences

### Front-end (elements significatifs + securite + argumentation)
- [ ] Maquettes de l application responsive
- [ ] Schema d enchainement des maquettes
- [ ] Captures d ecran de l interface utilisateur (responsive)
- [ ] Extraits de code d interface utilisateur statique
- [ ] Extraits de code de la partie dynamique des interfaces

### Back-end (elements significatifs + securite + argumentation)
- [ ] Presentation de la BDD (schemas conceptuels, physique, scripts SQL)
- [ ] Extraits de code de composants metiers
- [ ] Extraits de code de composants d acces aux donnees
- [ ] Presentation des elements de securite de l application
- [ ] Jeu d essai fonctionnalite representative (entree, attendu, obtenu, ecarts)
- [ ] Veille securite: vulnerabilites, failles trouvees, corrections

### Structure du memoire (bonnes pratiques)
- [ ] Page de garde
- [ ] Remerciements
- [ ] Resume (150 a 250 mots)
- [ ] Table des matieres
- [ ] Deroule selon cahier des charges
- [ ] Conclusion
- [ ] Bibliographie
- [ ] Annexes

### Redaction (bonnes pratiques)
- [ ] Style clair et concis (pas de phrases trop longues)
- [ ] Typographie: police lisible, interlignes adequats, marges correctes
- [ ] Pages numerotees
- [ ] Titres et sous-titres clairs
- [ ] Bon equilibre redaction / illustrations / schemas
- [ ] Pas de fautes d orthographe ni de grammaire

### Finalisation
- [ ] Tableau competences RNCP -> preuves
- [ ] Couverture activite front-end complete
- [ ] Couverture activite back-end complete
- [ ] Pieces justificatives (captures, liens, attestations)
- [ ] Relecture finale
- [ ] Export final (PDF/ZIP selon besoin)

## Dossier Professionnel (DP) - 3e preparation non-technique

> Deadline: S-2 avant le passage du titre RNCP.

- [ ] Le dossier professionnel est present
- [ ] Le DP reprend le template du Ministere du Travail
- [ ] Le DP comprend des exemples de pratiques professionnelles et/ou projets Holberton/personnels
- [ ] Le DP comprend des documents illustrant la pratique professionnelle (captures, code, schemas)
- [ ] Ces documents sont bien renseignes dans le tableau
- [ ] Le DP couvre toutes les activites du titre (REAC DWWM: AT1 + AT2)
- [ ] Chaque exemple comprend description des taches et conditions (objectifs, realisations, conditions)
- [ ] Chaque exemple comprend description des moyens utilises (comment ?)
- [ ] Chaque exemple comprend description de la collaboration (avec qui ? comment ? pourquoi ?)
- [ ] Chaque exemple explique le contexte
- [ ] Attestation de formation: tous les champs completes (si applicable)
- [ ] Identite du candidat renseignee
- [ ] Parcours du candidat (formation, experience, competences)
- [ ] AT1 - Au moins 1 exemple pour chaque CP front-end (CP1, CP2, CP3)
- [ ] AT2 - Au moins 1 exemple pour chaque CP back-end (CP4, CP5, CP6)
- [ ] Titres / diplomes anterieurs renseignes
- [ ] Declaration sur l honneur completee et signee
- [ ] Projets GitHub accessibles le jour J
- [ ] Export PDF du DP envoye a l equipe campus
- [ ] Manual Review valide par l equipe campus

## Support de presentation orale - 4e preparation non-technique

> Deadline: S-1 avant le passage du titre RNCP.
> Note de synthese: plus demandee. Intro anglais: remplacee par questionnaire pro en anglais.

- [ ] Le support de presentation est present
- [ ] Contexte du projet couvert
- [ ] Cahier des charges / expression des besoins couvert
- [ ] Environnement technique couvert
- [ ] Realisations front-end couvertes (maquettes, interfaces, code)
- [ ] Realisations back-end couvertes (BDD, composants, acces donnees)
- [ ] Securite de l application couverte
- [ ] Tests / jeux d essai couverts
- [ ] Veille securite couverte
- [ ] Conclusion / bilan present
- [ ] Slides annexes preparees pour questions du jury
- [ ] Duree coherente (1-2 min par slide)
- [ ] Bonnes pratiques visuelles (lisibilite, equilibre texte/illustration)
- [ ] Slides epures (texte limite, mots cles, phrases courtes)
- [ ] Police lisible (taille 24-30, titre 40, police simple type Arial/Calibri)
- [ ] Contraste visuel (texte sombre sur fond clair ou inversement)
- [ ] Visuels pertinents (graphiques, tableaux, images)
- [ ] Equilibre texte / image sur les slides
- [ ] Nombre de diapositives adapte (10-15 pour 10-20 min, double pour 35 min)
- [ ] Animations visuelles tres legeres uniquement
- [ ] Contenu des slides a l essentiel (messages cles, pas d infos secondaires)
- [ ] Langage technique et precis adapte au jury
- [ ] Orthographe verifiee sur le support de presentation
- [ ] Continuite logique et transitions entre slides
- [ ] Support envoye au staff pour verification (S-1)

### Entretien final (20 min max)

- [ ] Preparation aux questions d introduction et mise en confiance
- [ ] Preparation aux questions sur la vision et comprehension du metier
- [ ] Preparation aux questions sur la representation de la fonction de developpeur
- [ ] Preparation aux questions sur les technologies et la veille
- [ ] Preparation aux questions sur la gestion de projet et qualite
- [ ] Preparation aux questions sur l aspect securite (outils, RGPD, failles)
- [ ] Tache 0 MR: le candidat decrit correctement le metier et ses differentes activites
- [ ] Tache 0 MR: le candidat connait les contraintes d exercices specifiques du metier
- [ ] Tache 0 MR: le candidat fait la difference entre les contraintes des differents secteurs d emploi
- [ ] Tache 0 MR: le candidat prend en compte les imperatifs de production de l entreprise
- [ ] Tache 0 MR: le candidat a une idee precise de sa position et de son role (entreprise et externe)
- [ ] Tache 1 MR: le candidat explique les specificites du metier et de chaque activite
- [ ] Tache 1 MR: le candidat est capable d enchainer les activites du titre RNCP
- [ ] Tache 1 MR: le candidat integre la dimension gestion de projet et qualite
- [ ] Tache 1 MR: le candidat integre la dimension securite tout au long du processus de developpement
- [ ] Tache 1 MR: le candidat apprehende les evolutions de chaque activite
- [ ] Tache 1 MR: le candidat tient compte des considerations environnementales
- [ ] Tache 2 MR: le candidat adopte une presentation et un comportement en coherence avec le metier
- [ ] Tache 2 MR: le candidat utilise un langage technique etendu, pertinent, clair et adapte (francais et anglais)
- [ ] Tache 2 MR: le candidat recherche des informations sur les sites francophones et anglophones
- [ ] Tache 2 MR: le candidat montre son implication et sa motivation pour le metier
